# Scribe Dictation — maintainer notes for agents

Push-to-talk dictation via ElevenLabs Scribe v2. Three moving parts, two platforms:

| Part | File | Role |
|------|------|------|
| Engine (shared) | `realtime/scribe_stream.py` | mic (sox) → realtime WS → finalized lines on stdout / `--out-file` |
| mac glue | `init.lua` (Hammerspoon) | hotkeys, paste, menu-bar dot, settings via `hs.settings` |
| Windows glue | `windows/scribe.ahk` (AutoHotkey v2) | hotkeys, paste, tray dot, settings via `config.ini` |

Recording (batch) mode is separate: mac uses curl → REST in `init.lua`; Windows uses
`windows/scribe_batch.py`.

## Cross-platform invariants — keep both sides in lockstep

Any engine change lands on both platforms for free; any *behavioral* glue change must
be mirrored in the other glue. Current parity contract:

- **States/dots**: gray idle · green realtime · red recording · amber processing
  (mac `DOTS` in init.lua; Windows `icon-*.ico`, same RGB values — `icon-work.ico`
  was generated by recoloring `icon-rec.ico`'s pixels, keeping its alpha channel).
- **Stop is graceful** (`--stop-file` handshake): glue creates the stop file → engine
  stops sox, force-commits the tail, emits it, exits 0 → glue cleans up on exit.
  Amber while draining; second hotkey press force-kills; 8s watchdog both sides.
- **Paste discipline**: NEVER change the clipboard while a paste is still in flight.
  mac: serialized queue in `rtPaste`/`rtDrain` (violating this = last line pasted N
  times, earlier lines lost — the worst historical bug here). Windows: `PollStream`
  is `Critical` + `PasteText` is synchronous, and deliberately does NOT save/restore
  the old clipboard (restoring 120 ms after ^v races slow apps into pasting stale
  content).
- **Error contract**: engine prints one-line `SCRIBE-ERR <msg>` to stderr; both glues
  surface exactly that line and nothing else (sox failures are tagged this way too).
- **Segment spacing**: the engine strips each segment's whitespace, so BOTH glues
  append one trailing space per pasted line — without it English segments glue
  together (`there.How`).

## ElevenLabs realtime API facts (verified live, 2026-07)

- `committed_transcript_with_timestamps` is sent even without `include_timestamps`,
  but its `words` array is **empty** unless `include_timestamps=true` (the engine
  sets it only when `--timer` is on). Output paths read this message; plain
  `committed_transcript` is ignored as a duplicate.
- Word `start`/`end` are seconds, **absolute across the whole session** (not
  per-segment). The pacing timer and its `last_end` dedup depend on this.
- Force-flush = `input_audio_chunk` with `"commit": true` (empty audio OK); works in
  VAD mode, ~200 ms. A standalone `{"message_type": "commit"}` is **rejected**.
- Under `commit_strategy=vad`, audio cut mid-speech is *never* committed on its own —
  that's why graceful stop must send the commit flag.

## Platform gotchas

- **AutoHotkey v2**: a `;` is a comment ONLY when preceded by whitespace.
  `foo(); bar()` is a LOAD-TIME ERROR (the whole tray app fails to start). One
  statement per line. There's no AHK runtime on macOS — before committing, at least
  scan for attached semicolons and check brace balance; real verification needs a
  Windows box.
- **Hammerspoon deploy**: `~/.hammerspoon/init.lua` is a **copy** (install.sh), not a
  symlink. After editing repo `init.lua`: `cp init.lua ~/.hammerspoon/` then restart
  Hammerspoon (`osascript -e 'quit app "Hammerspoon"'; open -a Hammerspoon`).
  `hs -c "hs.reload()"` does NOT work — the config doesn't load `hs.ipc`.
- The engine runs from the repo working tree directly (`M.pyProject`), so engine
  edits are live on the next stream start with no deploy step.
- In Lua, check declaration order when adding shared upvalues — a use before the
  `local` silently becomes a global, and `luac -p` won't catch it.

## Testing without speaking into a mic

- Syntax: `luac -p init.lua` · `.venv/bin/python -m py_compile realtime/scribe_stream.py`.
- Engine unit-level: fake the WS with an async iterator of JSON frames and call
  `receive()` directly (see git history / scratchpad `test_receive.py` pattern).
- Engine end-to-end against the live API: generate speech with macOS
  `say -o x.aiff "... [[slnc 2000]] ..."` (the pauses trigger VAD commits), convert
  with sox to raw 16k mono PCM, then pipe it **at realtime pace** into the engine
  with `SCRIBE_SOX_INPUT="-t raw -r 16000 -b 16 -c 1 -e signed-integer -"` (sox then
  reads the engine's stdin). Touch the stop file mid-feed to test the drain.
  API key: `security find-generic-password -s elevenlabs-api -w`. Costs ~pennies.
- A stop mid-sentence should paste the tail within ~2 s and exit 0; a stop after a
  pause should exit in ~1.5 s with no duplicate of already-pasted text.

## Conventions

- Commit messages follow the existing history: imperative subject, a body that
  explains *why* (root cause → fix), no bullets-only bodies.
- User-facing settings persist outside the repo (hs.settings / config.ini) so
  `git pull` never resets them; `M.*` / CONFIG values are first-run defaults only.
- Keep README.md, windows/README.md, and realtime/README.md in sync with behavior
  changes (hotkeys, dots, stop semantics, timer options).
